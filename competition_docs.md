## AI 輕量化說明

### 採用輕量化 AI 之理由

1. **Multi-Agent 核心架構**
   - OpenClaw 擔任 Companion Agent 與 Evaluator Agent 的協調樞紐
   - 支援 WhatsApp/Telegram/Discord 多管道整合
   - 是整個應用系統的核心靈魂

2. **低成本部署**
   - 使用開源技術堆疊（OpenClaw + Flask + SQLite）
   - 無需購買昂貴的雲端服務

3. **節能環保**
   - 本地部署減少資料傳輸能源消耗
   - SQLite 資料庫輕量高效

4. **雲端彈性**
   - 串接外部 LLM API，無需自建 AI 運算環境
   - 可靈活選擇不同 LLM 服務商

### 輕量化技術應用

| 技術 | 應用場景 | 輕量化效益 |
|------|----------|------------|
| **WhatsApp** | 使用者介面 | 無需開發 App，使用戶已有的通訊軟體，零學習成本 |
| **OpenClaw** | Multi-Agent 核心架構 | 擔任 Companion Agent 與 Evaluator Agent 的協調樞紐，支援多管道整合，是整個應用系統的核心靈魂 |
| **Flask** | Web 伺服器 | 輕量 Python 框架，啟動快速，資源佔用低 |
| **SQLite** | 資料庫 | 嵌入式資料庫，無需伺服器程序，單一檔案儲存 |

---

### 為何選擇輕量化架構？

1. **降低導入門檻**：傳統 AI 系統需要 GPU 伺服器，成本高昂。本系統一般電腦即可運行。

2. **維護簡單**：無需專業 IT 團隊維護，降低人力成本。

3. **彈性擴展**：透過 API 串接不同 LLM 服務，可視需求選擇付費或免費方案。

4. **快速部署**：從安裝到上線只需 30 分鐘，適合時間緊迫的專案。
